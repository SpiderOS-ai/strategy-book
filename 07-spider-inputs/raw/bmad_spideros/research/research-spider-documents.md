## Résumé Exécutif

Spider Documents se positionne dans un espace unique combinant :
- Notes stratégiques issues de découvertes client (calls)
- Propositions commerciales (propales) automatisées
- Génération de slides depuis les calls
- Intégration d'un design system déductible depuis URL

**Insight clé :** Le marché est divisé entre outils de création (Gamma, Beautiful.ai) et outils de tracking (Qwilr, Proposify, PandaDoc). **Aucun ne fait les deux excellemment.** C'est l'opportunité de Spider Documents.

### Proposition de Valeur

> "De la conversation au closing : Spider Documents transforme vos calls clients en documents de vente personnalisés en quelques minutes."

### Gap du Marché

**Personne ne fait la chaîne complète :**
```
Call Noota/Aircall → Transcription → Analyse Spider People (contexte client)
                                    ↓
                          Note stratégique structurée
                                    ↓
                          Slides avec design system
                                    ↓
                          Lien web trackable
```

## Analyse UX des Concurrents

### Gamma — UX de Création

**Workflow :**
1. Entrer un prompt ou coller du texte
2. L'IA génère une présentation complète
3. Sélection d'un thème
4. Édition via interface type Notion (slash commands)

**Points forts :**
- Génération très rapide (quelques secondes)
- "Skip the blank page"
- Export multi-format

**Points faibles :**
- Interface dense : slash commands + chatbot + toolbar + mini-menus
- Customisation limitée : "prisonnier" des thèmes et blocks
- **Export PPT peu fiable — nécessite cleanup significatif**

**Notes :** 4.5/5 utilisation, 3/5 Trustpilot (support critique)

### Beautiful.ai — UX de Création

**Workflow :**
1. Input contenu ou sélection de template
2. "Smart Slides" adaptent automatiquement le design
3. Texte auto-resize, charts scale dynamiquement

**Points forts :**
- Zero formatting : tout s'aligne automatiquement
- Interface clean, pas de clutter
- Collaboration temps réel

**Points faibles :**
- **Layouts rigides — impossible de déplacer librement les éléments**
- Pas de "custom canvas mode"
- Frustrant pour designers habitués à Figma/Adobe

### Synthèse Comparative UX

| Aspect | Gamma | Beautiful.ai | Pitch |
|--------|-------|--------------|-------|
| **Vitesse de création** | +++ | ++ | ++ |
| **Contrôle design** | + | -- | ++ |
| **Simplicité interface** | ++ | +++ | ++ |
| **Qualité génération IA** | +++ | + | ++ |
| **Export fiable** | + | +++ | ++ |
| **Collaboration** | +++ | ++ | +++ |

- Gamma = génération rapide mais exports médiocres

## Templates Contraints - Approche Technique

### Le Problème du "Fool-Proof" Design

**Risque identifié dans le brainstorming :** "Le commercial peut quand même faire moche"

### Approches du Marché

| Approche | Exemple | Avantage | Limite |
|----------|---------|----------|--------|
| **Smart Slides** | Beautiful.ai | Design garanti | Utilisateur frustré |
| **Block-based** | Gamma/Notion | Plus flexible | Risque d'inconsistance |
| **Design System + Constraints** | IBM Carbon | Best of both | Plus complexe à implémenter |

### Recommandation pour Spider Documents

**Approche hybride :**

1. **Structure contrainte** — Sections pré-définies selon le type de document (note stratégique = 12 sections fixes)
2. **Blocs éditables mais bornés** — Texte modifiable, mais pas de changement de layout
3. **Design system automatique** — Couleurs/fonts extraits par Spider Content
4. **Édition par prompt** — "Refais cette section" plutôt que drag-and-drop libre

**Bénéfice :** Le commercial peut personnaliser le contenu sans casser le design.

## Tracking Web Avancé

### Leaders du Tracking de Propositions

#### Qwilr — Le plus avancé

- Analytics page-by-page
- **Time Spent section** — temps passé par bloc (liste ET pie chart)
- Identification des viewers (nom, role, entreprise)
- Tracking des clics sur liens/CTA
- Alertes temps réel quand document ouvert

**Insight :** "Proposals with engagement analytics achieve ~45% response rate vs 24% without tracking"

#### Proposify — Section-by-Section

- Tracking opens, views, time on page
- **Section-by-section analytics** — "what they read, how long they stayed, where they dropped off"
- Engagement trends

**Différenciation :** "Qwilr shows you when someone opens. Proposify shows what they read."

#### GetAccept — Deal Score

- Tracking temps par section
- Détection de nouveaux stakeholders
- **"Deal Score"** — probabilité de signature basée sur l'engagement
- Live chat dans le document

**Prix :** $49/user/mois (min 5 users)

### Technologies de Heatmap Web

| Outil | Type | Prix | Points forts |
|-------|------|------|--------------|
| **Microsoft Clarity** | Heatmap + Session recording | Gratuit | Unlimited traffic, scroll maps |
| **Hotjar** | Heatmap + Feedback | Freemium | Clicks, scrolls, movements |
| **Mouseflow** | Heatmap avancé | Payant | Attention heatmap (temps par zone) |

### Recommandation Tracking Spider Documents

**MVP Tracking :**
1. Vues (qui, quand, combien de fois)
2. Temps par section (comme Qwilr)
3. Alertes temps réel (document ouvert)
4. Partage détecté (nouveaux viewers)

**V2 Tracking :**
1. Heatmap attention
2. Scroll depth par section
3. Retours en arrière (sections relues)
4. Deal Score prédictif

**Avantage Web-Only :** Format web-only garantit le tracking. Pas de PDF qui échappe.

## Analyse SWOT

### Forces (Strengths)

1. **Flux unique Discovery-to-Document** — Aucun concurrent ne propose la chaîne complète call → notes → document → slides
2. **Intégration Spider People** — Données clients directement injectées
3. **Design system déductible** — Import URL pour déduire automatiquement la charte graphique
4. **Positionnement vertical sales/consulting** — Focus clair vs outils généralistes
5. **Fusion notes + propales** — Une seule source de vérité post-call

### Faiblesses (Weaknesses)

1. **Marque inconnue** — Compétition contre Gamma (10M+ utilisateurs)
2. **Coût de développement** — Multiple fonctionnalités IA à construire
3. **Dépendance à Spider People** — Nécessite adoption de l'écosystème
4. **Templates limités au lancement** — vs bibliothèques massives de Canva/Notion

### Opportunités (Opportunities)

1. **Abandon de Tome slides** — Vide sur le marché storytelling
2. **Fatigue des outils fragmentés** — Entreprises cherchent solutions intégrées
3. **Croissance explosive** — +25% TCAC présentations IA, +33% IDP
4. **Agents IA en essor** — 72% des entreprises adoptent d'ici 2026
5. **Marché francophone sous-exploité** — Peu d'outils IA natifs en français

### Menaces (Threats)

1. **Microsoft Copilot** — Intégration native M365, massive base installée
2. **Google Gemini** — Intégration native Google Workspace
3. **Consolidation du marché** — Acquisitions potentielles (Canva, Adobe)
4. **Course à l'IA** — Features copiées rapidement
5. **40% des projets agents annulés (Gartner)** — Risque technique

## Key Insights Consolidés

### Insight 1 : Le marché est divisé — c'est une opportunité
Outils de création vs outils de tracking. Aucun ne fait les deux bien.

### Insight 2 : L'UX "contrainte" est clivante mais stratégique
Beautiful.ai frustre les designers mais ravit les non-designers.

### Insight 3 : Le tracking section-by-section augmente les conversions de +21%

### Insight 4 : Personne ne fait "transcription → slides" de bout en bout
C'est le différenciateur unique de Spider Documents.

### Insight 5 : L'export PPT est un problème universel
La stratégie "web-only" évite ce problème ET garantit le tracking. Win-win.

### Insight 6 : Le gain de temps est énorme (17-25h → <4min)
"10 min de validation au lieu de 3h de rédaction" est crédible et compelling.
