4.1 Les 5 Pourquoi sur chaque these majeure

T1 : Le tacite est le dernier tresor

Cause racine : Le tacite europeen est culturellement invisible pour les builders tech parce qu'il est, par nature, ce qui ne s'ecrit pas. Le LLM est le premier outil capable de le capturer par observation passive. MAIS : cela suppose que la capture passive fonctionne en contexte PME (non prouve) et que la valeur capturee est perceptible par le dirigeant (non prouve).

Implication GTM : Spider ne peut pas vendre "on capture votre tacite." Spider doit vendre un benefice tangible (gain de temps, argent recupere) et capturer le tacite en sous-produit. Le pitch "dernier tresor" est un narratif investisseur, pas un pitch client.

T8 : Le code est mort, le service est le produit

Cause racine : Le Service-as-Software fonctionne SI ET SEULEMENT SI l'IA reduit reellement le cout marginal par client dans le temps. Bench a echoue parce que les humains restaient necessaires et que le cout ne baissait pas.

Kill criteria H6 : si les heures-humaines par client ne baissent pas apres 6 mois, pivoter. Spider est Bench avec un meilleur marketing.

T9 : Le Shogunat Digital

Cause racine : T9 est une metaphore puissante pour le pitch investisseur. Ce n'est pas un mecanisme operationnel. Le passage de "assistant utile" a "shogun qui gouverne" suppose (a) une IA quasi-infaillible, (b) un dirigeant passif, (c) un cout de sortie suffisant. Les trois sont non prouves.

Implication GTM : Ne JAMAIS utiliser la metaphore du Shogunat devant un client PME. Le dirigeant veut un assistant fiable, pas un shogun. Vendre la tranquillite, pas le pouvoir.
