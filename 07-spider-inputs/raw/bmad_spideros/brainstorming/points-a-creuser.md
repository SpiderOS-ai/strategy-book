# Points a creuser — Cross-modules

Ce fichier centralise tous les sujets mentionnes pendant les brainstormings qui meritent d'etre approfondis.

---

## Spider People

- [ ] **Playbook** — Concept central pas assez defini. C'est quoi exactement? Process commercial, etapes, cibles, offres, questions a poser? Comment on le visualise? Comment on le configure?
- [ ] **Score de connaissance** — Dimensions a tracker pour vraiment "connaitre" quelqu'un (MBTI, DISC, CIA, psychologie). Recherche necessaire.
- [ ] **Fiabilite de la data** — Comment atteindre >90% sans trop de validations? Scrapping? Predictif?

---

## Spider Documents

- [ ] **Template evolutif** — Template qui evolue avec des sections qui se rajoutent. Vue du socle commun de toutes les sections. Lead marketing peut valider/organiser.
- [ ] **Imbrication des modules** — Note + Slides generees ensemble. Pre-brief avant decouverte + Note finale apres. Les questions du playbook generent le livrable.
- [ ] **Templates par type de livrable** — User peut definir exactement ce qu'il veut pour chaque type (note strat, audit, propale, etc.)
- [ ] **Tracking avance** — Temps passe par section, retours en arriere, ce qui est lu vs survole
- [ ] **Feedback prospect integre** — Le prospect peut commenter directement sur la note, ca alimente Spider People
- [ ] **Edition contrainte mais flexible** — Pas de freestyle design, mais modification du texte possible + feedback focus par section ("cette section ne me plait pas")
- [ ] **Relecture auto** — Verification fautes/coherence avant envoi (pas P0)
