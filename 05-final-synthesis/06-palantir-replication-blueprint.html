01 The Big Picture
02 Architecture — What to Build
The Ontology is the moat. Every other layer has 50-95% open-source equivalents. The Operational Ontology — a live, governed, action-capable business model — has ~30% replicability. No OSS project does this.

6 Primitives

  • Objects — entities (Client, Order…)
  • Properties — attributes
  • Links — relationships
  • Actions — operations + writeback
  • Interfaces — abstract types
  • Functions — computed values

3 Layers

  • Semantic — WHAT: data model, business entities
  • Kinetic — HOW: real-time flows, actions, writeback
  • Dynamic — GUARDRAILS: RBAC, audit, provenance

Key Decisions

  • PostgreSQL JSONB + pgvector as backbone
  • User edits = first-class, append-only log
  • Optimistic concurrency on Actions
  • Schema evolution via manual remapping
  • "Copy the contracts, not the code"
03 Delivery — How to Deploy

FDE — Forward Deployed Engineer

Not a consultant. A "startup CTO" embedded at the client site. Builds production software on the platform. 4-5 per customer in Acquire phase. Codename "Delta". $171-415K total comp entry level. Insights feed back into platform R&D.

Deployment Strategist — "Echo"

Ex-McKinsey/BCG. Hybrid: consultant + sales engineer + PM. Shapes the deal, identifies highest-impact use cases, manages C-level relationships. 1-2 per bootcamp.

The Flywheel Effect

FDE work on client problems → discovers platform gaps → improvements shipped to product → benefits all customers. This is why FDE costs are classified as R&D, not COGS. Revenue/employee: $1.01M.

03d AIFD — How AI Changed Everything
"AIFD" = AI-First Deployment. Not a formal product — it's how AIP accelerates FDE work. SAP migrations that took years now take weeks. An "army of SI consultants" replaced by two humans + "an army of AI FTEs."

AI Does Well

  • Schema/ontology mapping from data sources
  • Pipeline code generation (80%+ boilerplate)
  • Workshop app scaffolding from NL descriptions
  • Migration plan generation (60 seconds)

AI Assists

  • Data quality assessment
  • Link inference from data structure
  • Application UI refinement
  • Documentation generation

Still Human

  • Client relationship management
  • Complex business logic / domain expertise
  • Security architecture (classified)
  • Change management & adoption
  • Data access politics
Jevons Paradox in action: Despite AI making each FDE 5-15x more productive, Palantir hired MORE people (3,838 → 4,414 headcount 2023→2025). AI is a scaling tool, not a cost-cutting tool. Each FDE handles more engagements → more revenue → hire more.

The core insight: Palantir proved the Ontology model works. Spider's bet is that the same architecture — simplified, templated, AI-automated — can serve the 50-100K European companies that have an ERP + €10-100M revenue but zero data infrastructure.
Counter-thesis to internalize: 83% of EU SMBs use Excel for analytics. 80% of AI projects fail (RAND). APEX benchmark: 29.8% best score. The gap isn't technology — it's organizational readiness. Spider must solve adoption, not just architecture.
